MINIMUM NUMBER OF JUMPS

ABORDAGEM UTILIZANDO PROGRAMAÇÃO DINÂMICA

DESCRIÇÃO 

O problema

Dada uma sequência de números inteiros, em que cada número representa a capacidade máxima de posições que podem ser “saltadas” (Jump) a partir daquele elemento da sequência, questiona-se: Qual o menor número de saltos que devem ser executados? A fim de chegar ao último número da sequência partindo do primeiro. Uma observação importante é que se alguma posição contiver o valor 0, não é possível executar salto algum partindo daquela posição.  

Um caso exemplo pode ser ilustrado a partir desta sequência de 10 números inteiros: 
As valores associados a cada posição revelam quantas casas podem ser saltadas a partir de cada um deles. No caso do primeiro elemento, seu alcance se estende a ambos o segundo e terceiro elementos. 
Já no caso do segundo elemento, seu alcance se estende por 3 outros elementos: terceiro, quarto e quinto. 
A mesma pergunta se mantém. Qual o número mínimo de saltos para alcançar o elemento final? A resposta para este exemplo é 4. 
Como pode ser observado no caso acima, nem sempre o melhor caminho é sempre saltar até a maior distância viável a partir daquele elemento, mas analisar caso a caso para entender o salto ótimo. Existem diversas formas de resolver este problema e entender as diferentes abordagens permite compreender suas otimizações associadas. Para a resolução deste problema este documento discorre sobre sua implementação utilizando programação dinâmica. 



CASE

InsperTrans - Case

A empresa de transporte InsperTrans está desenvolvendo uma plataforma online para auxiliar seus usuários com a utilização de seu sistema de transporte.

As linhas da InsperTrans possuem uma peculiaridade. Para otimizar as transições rápidas nas paradas, os ônibus somente embarcam passageiros em seus pontos iniciais. Garantindo que todos os passageiros sejam embarcados e viajem com conforto. A cada entrada em um ônibus o usuário deve utilizar um novo ticket.

O objetivo de sua equipe é desenvolver um algoritmo que entrega ao usuário o número de tickets que devem ser comprados para ir de um ponto até outro, bem como os pontos os quais o usuário deve trocar de ônibus.



RESOLUÇÂO

Abordagem Dinâmica

A abordagem utilizando programação dinâmica (Dynamic Programming - DP) consiste da utilização de artifícios programacionais, que visam o incremento na eficiência de execução. Tratando das o modelo de multiplas chamadas para o mesmo fator e o armazenamento infeciênte da solução como fatores maléficos à eficiência do algoritmo recursivo.

O problema será resolvido através da utilização de técnicas de memoização. Basicamente o código armazenará os subproblemas sendo resolvidos em uma estrutura análoga a um cache, permitindo que o cálculo não tenha que ser feito novamente repetidas vezes. Associado a essa manobra, o programa utilizando DP faz uso de uma estrutura de subproblemas mais otimizada, dessa forma garantindo complexidade quadrática no pior caso O(n^2).

O que o algoritmo faz é guardar os valores visando não gerar redundância. O processo utilza um array/lista auxiliar para a anotação do fator de MNJ até o indice de análise, ou seja, sabemos o valor do MNJ para a posição de índice 1. 0 -> 1 é a única opção. Portanto, só pode ser 1. Assim, podemos concluir que o MNJ até a a posição de índice 2, deve ser o mínimo entre as opções possíveis. 0 -> 2 ou 1 -> 2 (Sabendo que 0 -> 2 é uma opção válida), apenas deve-se avaliar 0 MNJ até o ponto somado de 1 (já que um novo salto está sendo executado). Em uma comparação 0 -> 2 é o salto que levaria à contagem correta de MNJ do item de posição 2, MNJ: 1.

Este conceito representa a utilização dos subproblemas unidos, uma vez que o problema maior: MNJ(0 -> n), é subdividido em MNJ(0 -> 0+1), (0 -> 0+2), (0 -> 0+(n-1)). Sendo unidos ao calcular o item seguinte a sequência.

6  int minJumps(int arr[], int n)
7  {
8     int *jumps = new int[n];  // jumps[n-1] will hold the result
9     int i, j;
10
11    if (n == 0 || arr[0] == 0)
12       return INT_MAX;
13
14    jumps[0] = 0;
15
16    // Find the minimum number of jumps to reach arr[i]
17    // from arr[0], and assign this value to jumps[i]
18    for (i = 1; i < n; i++)
19    {
20        jumps[i] = INT_MAX;
21        for (j = 0; j < i; j++)
22        {
23            if (i <= j + arr[j] && jumps[j] != INT_MAX)
24            {
25                 jumps[i] = std::min(jumps[i], jumps[j] + 1);
26                 break;
27            }
28        }
29    }
30    return jumps[n-1];
31 }
